Advanced BI (Power BI) — Calc Groups, Parameters, RLS

Deep-dive build focused on Power BI modeling & UX patterns: calculation groups for time intelligence, field/what-if parameters for dynamic analysis, RLS for secure multi-tenant views, and bookmarks for guided stories.

View Repo Download PBIX PDF Case Study

Business Questions

  • How do KPIs change across time grains (MTD/QTD/YTD/R12) without duplicating measures?
  • Can analysts pivot dimensions on the fly (field parameters) for ad-hoc exploration?
  • How do we keep data secure per user (RLS) while preserving performance?

Data & Model

Star schema with a robust Date table. Calculation group for time intelligence; disconnected tables for parameters and what-if scenarios. Roles for RLS.

Measures (examples):
• Total Amount := SUM(fact_finance[amount])
• Amount YTD := TOTALYTD([Total Amount], 'dim_date'[full_date])
• Amount R12 := CALCULATE([Total Amount], DATESINPERIOD('dim_date'[full_date], MAX('dim_date'[full_date]), -12, MONTH))

Key Features

  • Calculation Groups for dynamic MTD/QTD/YTD/R12 without measure explosion
  • Field & What-If Parameters to swap axes/measures and run scenario analysis
  • RLS by entity/region for secure, personalized dashboards
  • Bookmarks to create guided stories and toggle views

Insights & Outcomes

  • Reduced maintenance (single measure logic reused via calc groups)
  • Self-service analytics with safe guardrails
  • Security-compliant distribution to multiple stakeholders
